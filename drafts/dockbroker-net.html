Networking in dockbroker

// under Docker Go HPC peer-to-peer distributed in Tech    // share on Twitter

All connections will be permanently open connections via WebSockets. In this way, not all nodes need to be on an accessible network. It is enough when a broker on a private network connects to a public broker to establish a two-way communication. Number of open connection in Go should not be a problem as it would be with a full Apache given that Go does not create a separate thread for every connection. My estimate is that permanent connections for thousands of machines are fine; to be tested.

Zero capability brokers

Zero capability refers to their compute and data store capabilities. They are still part of the network of brokers and can refer clients to other brokers. One such broker can be used as a public server to redirect requests from private networks.
